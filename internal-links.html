A device whose primary purpose is Routing

  • Routers are connected to a network.
  • They have an IP address and a MAC address.
  • They forward packets not destined to themselves (unlike hosts).
  • When a router receives packets with an unknown Destination IP, packets are dropped.
  • Routers maintain a map of all the Networks they know about:
    • Routing Table - A routing table is populated by three methods:
      • Directly Connected: Routes for attached networks.
      • Static Routing: Routes manually provided by an Admin.
      • Dynamic Routing: Routers automatically learn from other routers.
  • Routers also have ARP Tables - mapping of L3 to L2 addresses:
    • Everything with an IP address has an ARP Table.

Network Protocols

A protocol is a set of rules and messages that form an Internet standard.

  • ARP - Address Resolution Protocol (RFC 826): Resolves IP to MAC mapping using ARP Requests/Responses.
  • FTP - File Transfer Protocol
  • SMTP - Simple Mail Transfer Protocol
  • HTTP - Hyper Text Transfer Protocol
  • SSL - Secure Sockets Layer
  • TLS - Transport Layer Security
  • HTTPS - HTTP secured with SSL/TLS

IP Addressing and Subnetting

  • An IP Address is the identity of each host.
  • An IP address consists of 32 Bits represented by four octets, each ranging from 0 to 255.
  • IP Addresses are hierarchically assigned, allowing for Subnetting.

Network Classes

Network Class Range Subnet Mask
A 1.0.0.0 - 126.255.255.255 255.0.0.0
B 128.0.0.0 - 191.255.0.0 255.255.0.0
C 192.0.0.0 - 223.255.255.0 255.255.255.0
D 224.0.0.0 - 239.255.255.255 N/A
E 240.0.0.0 - 255.255.255.255 N/A

The job of the subnet mask is to dictate how many devices can exist on a network. For example, with an IP address of 192.168.1.0 and a subnet mask of 255.255.255.0, we have access to 255 IP addresses, with three reserved as follows:

  • 192.168.1.0: Network Address
  • 192.168.1.1: Gateway
  • 192.168.1.255: Broadcast Address
